# Binary Outlook Portfolio Page Blueprint

## Purpose of the Webpage

This page will be a public technical homepage for **Binary Outlook**.

The goal is to make the visitor understand, quickly and clearly:

- what Binary Outlook is interested in,
- what kinds of technical projects Binary Outlook builds,
- which areas Binary Outlook is currently exploring,
- where to inspect the work,
- and how to connect.

This should feel like a serious computing portfolio rather than a personal diary or generic resume page. The page should communicate curiosity, technical direction, and project taste.

## Site Structure Decision

For the first version, one webpage is enough.

The current content fits well into a single-page portfolio because the main job is introduction and credibility. A visitor should not need to click through many pages just to understand the work.

Later, the site can expand into separate pages if needed:

- `projects.md` for longer project writeups,
- `notes.md` for technical learning notes,
- `research.md` if research work becomes a major part of the profile,
- `writing.md` if Binary Outlook starts publishing essays or technical reflections.

For now, the homepage should include all essential content.

## Tone and Writing Rules

The writing should be:

- clear,
- technically serious,
- curious,
- direct,
- not exaggerated,
- not stuffed with buzzwords,
- not written like a corporate landing page.

Avoid phrases like:

- "passionate innovator"
- "revolutionizing the future"
- "AI-powered everything"
- "visionary technologist"
- "world-class"

Prefer phrases like:

- "building practical systems"
- "exploring technical ideas"
- "turning concepts into tools"
- "learning with intent"
- "project-driven exploration"

## Visual Direction for Later HTML/CSS/JS

This is not the final design yet, but the page should probably feel:

- precise,
- modern,
- technical,
- calm,
- slightly editorial,
- project-focused.

Avoid making the page look like a generic startup landing page.

Avoid oversized marketing slogans.

Avoid heavy purple-blue gradients, glowing dark-mode effects, and repeated card grids.

The design should let projects and technical direction carry the page.
