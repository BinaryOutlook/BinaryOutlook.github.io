## Design Context

### Users
Visitors are people evaluating the site's technical direction: recruiters, collaborators, classmates, hackathon teammates, and engineers scanning for credible project work.

### Brand Personality
Professional, curious, geometric. The page should feel like a serious programming portfolio with a clear technical point of view, not a generic resume page or casual diary.

### Aesthetic Direction
Use Bauhaus-inspired structure: hard-edged geometry, primary accent colors, visible grids, asymmetric balance, and direct typography. Keep the result polished and technical rather than playful or decorative.

### Design Principles
- Keep Binary Outlook as the public brand/account name, but use first-person body copy for a more consistent personal voice.
- Prefer structured, native UI over pasted-in third-party images.
- Use geometry and color as system signals, not decoration.
- Make project and GitHub sections easy to scan quickly.
- Preserve accessibility, reduced-motion behavior, and responsive layouts.
